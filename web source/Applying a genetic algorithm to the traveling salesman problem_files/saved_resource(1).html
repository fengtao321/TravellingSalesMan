                                                                                                                                                                              Asherigo Vespucci • 2 years ago

                                                                                                                                                                              Your chromosomes/GA should reflect the fact that the sum of the shortest links makes the shortest path. Alternatively, a penalty for links forming an acute angle at a given vertex; or that the convex hull has largest enclosed area. Ideally the chromosomes should be represented as a 2D objects for a planar graph.

                                                                                                                                                                              Compare your solution -- distance 940:
                                                                                                                                                                              con940 = {{60, 200}, {20, 160}, {40, 120}, {60, 80}, {20, 40}, {20, 20}, {60,
                                                                                                                                                                              20}, {100, 40}, {160, 20}, {200, 40}, {180, 60}, {120, 80}, {140,
                                                                                                                                                                              140}, {180, 100}, {200, 160}, {180, 200}, {140, 180}, {100,
                                                                                                                                                                              120}, {100, 160}, {80, 180}, {60, 200}}

                                                                                                                                                                              to the following configuration -- distance 863:
                                                                                                                                                                              con863 = {{140, 180}, {180, 200}, {200, 160}, {180, 100}, {180, 60}, {200,
                                                                                                                                                                              40}, {160, 20}, {120, 80}, {100, 40}, {60, 20}, {20, 20}, {20,
                                                                                                                                                                              40}, {60, 80}, {40, 120}, {20, 160}, {60, 200}, {80, 180}, {100,
                                                                                                                                                                              160}, {100, 120}, {140, 140}, {140, 180}}

                                                                                                                                                                              However, for a "1D problem" such as an ordered sequential configuration as in a grid of cities e.g.
                                                                                                                                                                              for(int i = 0; i < 5; i++){
                                                                                                                                                                              for(int j = 0; j < 5; j++){
                                                                                                                                                                              City city = new City((i+1)*10, (j+1)*10);
                                                                                                                                                                              TourManager.addCity(city);
                                                                                                                                                                              }
                                                                                                                                                                              };

                                                                                                                                                                              the GA algorithm can be tested against the optimal distance of a Hamiltonian solution -- 24*10+14 = 254. There is a need for one and only one 'kink' (diagonal) as in the following configuration:
                                                                                                                                                                              {{10, 40}, {10, 30}, {10, 20}, {10, 10}, {20, 10}, {30, 10}, {30,
                                                                                                                                                                              20}, {20, 20}, {20, 30}, {20, 40}, {30, 40}, {40, 40}, {30,
                                                                                                                                                                              30}, {40, 30}, {40, 20}, {40, 10}, {50, 10}, {50, 20}, {50,
                                                                                                                                                                              30}, {50, 40}, {50, 50}, {40, 50}, {30, 50}, {20, 50}, {10,
                                                                                                                                                                              50}} -- indeed the distance is 254. This solution can be attained in very few iterations.
